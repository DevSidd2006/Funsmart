## 🎨 SECTION VISUAL HIERARCHY

### Hero Section Emphasis Path:
```
1. Background image (establishes credibility)
   ↓
2. Headline (emotional hook)
   ↓
3. Subheading (clarification)
   ↓
4. Primary CTA (action)
```

### Program Card Emphasis Path:
```
1. Program name
   ↓
2. "Perfect for" (relatable)
   ↓
3. Key features (tangible benefits)
   ↓
4. Price (transparency)
   ↓
5. CTA button (action)
```

### Testimonial Emphasis Path:
```
1. Quote (emotional impact)
   ↓
2. Parent name (credibility)
   ↓
3. Child age (relatability)
```

## 🔄 USER SCROLL JOURNEY (Desktop)

| Scroll Depth | What happens | Parent's mindset |
|---|---|---|
| **0% (Hero)** | Curiosity created | "This sounds different..." |
| **20% (Stats)** | Trust signals shown | "They have momentum" |
| **35% (Positioning)** | Core idea clarified | "Ah, I get it now" |
| **50% (How it Works)** | Process simplified | "This is doable" |
| **65% (Programs)** | Choice offered | "Which one fits me?" |
| **75% (Stories)** | Emotional proof shown | "Real parents did this" |
| **85% (FAQs)** | Objections addressed | "My concern is handled" |
| **95% (Final CTA)** | Call to action | "I'm ready to book" |

## 🎯 DESIGN DECISIONS MATCHED TO STEMpedia + FunSmartism Positioning

| Decision | Rationale | STEMpedia Base |
|----------|-----------|---|
| **Hero with real image** | Build trust immediately via authenticity | ✓ Used real school photos |
| **Stats row** | Social proof (numbers work for B2C) | ✓ Used heavily |
| **Comparison table** | Clarify difference (key barrier) | ✗ Less emphasis |
| **Timeline/process** | Simplify complexity for parents | ✓ Used slightly differently |
| **Program cards side-by-side** | Easy comparison | ✓ Card-based approach |
| **Testimonials as cards** | Emotional connection > features | ✓ Heavy testimonials |
| **FAQ accordion** | Address specific parent fears | ✗ Less prominent |
| **Soft color palette** | Premium + calm (not tech-y) | ✗ Brighter in STEMpedia |
| **Observation focus** | Reinforce core message | ✗ Feature-focused |

## ⚡ INTERACTIVE ELEMENTS

### Hover States
- **Links**: Sage green (#4F8F7F) with underline
- **Buttons**: Darker sage (#3D6B65) with slight shadow lift
- **Cards**: Subtle shadow increase (0 → 8px), border accent color

### Active States
- **Program cards**: Border becomes sage, slight background shift
- **FAQ accordion**: Active header shows as darker

### Micro-animations
- **CTA button**: Pulse effect on page load (subtle)
- **Stats**: Number counter animation on scroll-into-view (1–2 seconds)
- **Testimonials**: Slide transition (carousel, 500ms ease)
